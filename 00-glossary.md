# 00 · Glossary

The vocabulary the rest of the documentation assumes. **This document links to nothing** — it is the
bottom of the graph.

Terms are grouped by what they describe, not alphabetised, because the definitions build on each other.

---

## Build structure

**Reactor** — the Maven multi-module build rooted at `bus-core-api/pom.xml`. It aggregates every module
and the deployable. Building the reactor builds everything, in dependency order.

**Module** — one Maven artifact under `modules/`, owning one bounded concern. It compiles to a jar. A
module that does not declare a dependency on another **physically cannot import it** — javac refuses
before any architecture rule runs. That is the primary enforcement; everything else is secondary.

**Deployable** — `services/bus-core`, the single Spring Boot application. All module jars are nested
inside it at package time, so the deployable really is one file.

**Role** — which job the deployable is doing, selected by Spring profile: `api` (HTTP surface), `worker`
(no HTTP; sweeps and consumers), `gateway` (edge). One artifact, three roles. The artifact names the
system; the profile names the role.

**Slice** — a *vertical* increment: entities, migrations, service, controller and config, shipped
together, such that something real works end to end. A module joins the deployable when its first slice
lands — not when the module is created.

---

## Module anatomy

**Published surface** — the types at a module's package root (`tz.co.otapp.buscore.<module>`). These, and
only these, may be imported by another module. Publishing is a deliberate act: you move a type up.

**Internal** — everything under `<module>/internal/`. Module-private. No other module may import from it,
and the build fails if one tries.

**Port** — an interface that crosses a module boundary.
- A **published port** is declared by the module that *implements* it, and called by others.
- An **inbound port** is declared by the module that *needs* it and implemented by the module that *owns*
  the data — which reverses the compile arrow. Used when the natural direction would create a cycle.

**Adapter** — a module implementing a port owned by another, usually wrapping something external (a
payment vendor, a tax authority). **Adapters point inward**: they depend on the module whose port they
implement, and nothing depends on them.

## Identity

**Principal** — the authenticated actor for the current request, as other modules see it. Carries a uid, a
kind, and derived authority. Authority is read from here and **never** from a request body.

**PrincipalType** — which kind of actor: staff, agent, or machine client. Three kinds, three login
ceremonies, three security models.

**Permission** — a `DOMAIN.ACTION` string (e.g. `VEHICLE.READ`) checked at a route. It answers *what* may
be done.

**Role** — a named bundle of permissions. A grouping device for granting; the runtime check is on the
permission, not the role.

**Operator scope** — which operators' rows the caller may see and touch. It answers *whose*. Independent
of permission: a platform administrator who reaches every operator's vehicles still needs `VEHICLE.READ`
to read one. **Both checks always run.**

**Tenancy** — the organisation a staff account belongs to: the platform, one operator tree, or a partner.
Fixed at provisioning. It is not a job function and not a level in the operator tree.

---

## Data

**id** — a module-private surrogate primary key (`bigint`). **It never crosses a module boundary**, never
appears in a published type's signature, and never appears in a URL.

**uid** — a `uuid`, generated by the database, that is the *only* handle allowed to cross a boundary. When
one module refers to another's row, it stores that row's uid.

**uid handle** — a stored uid with **no foreign key and no association**. The referenced row lives in
another module's tables; a constraint would weld the two schemas together and invert the dependency arrow.
The cost is that nothing detects a dangling handle, so it must fail closed where it is resolved.

**Migration** — a versioned SQL file owned by one module. The database schema is owned by migrations,
never by Hibernate DDL. Each module has its own migration history table.

**Outbox** — a table a module writes an event row into **inside its own business transaction**, so the
event and the change it describes commit together or not at all. A relay then publishes committed rows to
the broker.

---

## Enforcement

**ArchUnit rule** — a test that fails the build on a structural violation. These read **bytecode**, not
source or configuration, which is what lets them catch an import that the compiler permitted.

**The DAG** — the declared set of allowed module-to-module edges. It exists in two places that must agree:
each module's pom, and a map in the architecture tests. An edge in the pom but not the map is a dependency
nobody decided on; an edge in the map but not the pom is a lie.

**Fail closed** — on error or missing configuration, deny rather than allow. The default posture.

**Fail open** — on error, allow. Correct only for defence-in-depth layers whose absence must not take the
system down (a rate limiter backed by an optional cache).
